# 部署说明

当前项目的完整说明请优先查看根目录的 `README.md`。

## 当前发布方式

- 源码分支：`master`
- 构建方式：GitHub Actions
- 发布目标：GitHub Pages
- 自定义域名：`cnmarathon.cn`

## 当前正确流程

1. 在本地修改源码
2. 本地执行：

```powershell
npm.cmd run build
```

3. 提交并推送：

```powershell
git add .
git commit -m "你的提交说明"
git push origin master
```

4. GitHub Actions 自动构建并发布

## 不再使用的旧流程

以下方式已经废弃：

- `hexo deploy`
- 手动维护 `.deploy_git`
- 提交 `public/` 作为发布内容
- 依赖发布分支推送静态文件

## 说明

如果 `git push` 后没有新的 Action，大概率是因为没有新的 commit 被推上去。
